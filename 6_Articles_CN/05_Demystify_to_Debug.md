> 前四篇，我在学AI的过程中发现——这些东西我其实早就见过。
> 这一篇，我在动手跑算例的过程中发现——**有些问题还没人认真提过。**

先说一句公道话。AI在语言理解和代码生成方面确实有突破性进展，不可否认。但恰恰因为这些突破，人们的期望溢出到了所有方向——物理仿真、自动驾驶、通用机器人、AGI。冷静看：AGI的路远得很，工程AI的路也远得很。本篇聚焦物理AI——AI能不能理解和模拟物理世界？

做了20多年CAE仿真的人，看到一个新工具，第一反应不是"太厉害了"，第二反应是"精度够吗？收敛吗？网格无关性验证了吗？"——**这不是批评，是本能。**

带着这个本能，我在WSL2上搭建了完整的PhysicsNeMo工作空间（2025年10月-12月），用PyTorch Geometric从零复现MeshGraphNet的圆柱绕流训练和预测，跑通了FNO Darcy流示例，从PyG的Karate Club GCN入门一步步走到完整实验。数据集几个G磁盘先卡住，GPU显存不够CUDA Out of Memory，训练跑了58小时跟多物理场耦合一个量级——**这些坑和我在CAE里踩过的坑，长得几乎一模一样。** 更有意思的是，有些坑在传统CAE里已经有成熟的解决方案，但在物理AI这边好像还没人认真对待。训练脚本、loss曲线、预测对比图全部开源：**github.com/tiexinding/NeuralCAE-experiments**

## 附录A：四组对应关系的边界分析

前四篇建立了神经网络和CAE之间的四组核心对应关系。第四篇附录A是正面展开——证明"哪里一样"。这里反过来，聊每组对应关系的**边界**——搞清楚"哪里不一样"才能真正用起来。

### A.1 ResNet残差 ≈ Newton-Raphson迭代——但不完全是

| CAE迭代求解 | ResNet深度学习 |
|---|---|
| 当前解 x_n | 上一层输出（identity） |
| 残差修正 Δx_n | 残差块学到的 F(x) |
| Newton-Raphson迭代 | 逐层传播 |
| 收敛判据（残差→0） | 损失函数最小化 |

结构上几乎完美对应。但仔细一想：

**收敛性保证在哪？** Newton-Raphson有严格的收敛性理论——在初始值足够接近真解的条件下，二次收敛。我们知道什么时候该停（残差 < 容差），也知道不收敛时该做什么（减小增量步、换求解算法）。

ResNet呢？层数是固定的。不管问题解没解好，到了最后一层就是输出。没有"残差 < 容差就停止"这个概念。这在CAE工程师看来是不可接受的。你的求解器没有收敛判据？那你怎么知道结果可不可靠？

**有意思的方向：** 2018年陈天琦提出的Neural ODE，把ResNet解释为常微分方程的离散化——层数变成自适应的，"收敛"的概念也自然出现了。更进一步，Deep Equilibrium Models（DEQ）直接做到了"迭代到收敛再输出"——这才是Newton-Raphson的真正AI对应物。

**结论：** ResNet和Newton-Raphson之间的对应是结构类比，不是数学等价。但这个类比指向了一个真实的研究方向：给神经网络加上收敛性保证。

### A.2 GNN Message Passing = PNM节点守恒方程——这个是严格的

| PNM渗流模型 | GNN Message Passing |
|---|---|
| 孔隙节点 | 图节点 |
| 喉道连接 | 图边 |
| 守恒方程 ΣQ_ij=0 | 消息聚合 AGGREGATE |
| 达西定律/傅里叶定律 | MESSAGE函数 |
| 求解全局方程组 | 多轮message passing |

这一组我在NPM论文中做了严格数学证明：当消息函数取达西定律的形式时，GNN的前向传播精确等价于PNM的物理求解。

但即便是最严格的一组，也有值得深挖的地方：

**反方向的映射成立吗？** NPM证明了PNM→GNN的等价性。但反过来呢？任意GNN学到的消息函数，是否都能对应某个物理系统？如果可以，GNN就不仅是"黑盒预测器"，还能成为物理发现工具。

**GNN的"非物理解"问题。** GNN没有硬编码守恒律。训练出来的模型可能在统计意义上满足守恒（loss足够低），但在单个预测上可能违反守恒。传统CFD有全局质量/能量平衡检查，物理AI有吗？

### A.3 MeshGraphNets = 前处理→求解→后处理——流程一样，但细节里全是坑

| 传统CAE流程 | MeshGraphNets |
|---|---|
| HyperMesh/ANSA前处理 | Encoder编码器 |
| Abaqus/ANSYS求解器 | Processor（多轮message passing） |
| HyperView/ParaView后处理 | Decoder解码器 |

**网格无关性在哪？** 传统CAE最基本的验证步骤——把网格加密，看结果是否趋于收敛。MeshGraphNet的图节点和边呢？增加节点数、改变连接半径、增加MP轮数，预测结果会收敛吗？目前没有人系统回答过这些问题。

**Encoder ≠ 真正的前处理。** 传统前处理需要大量工程判断：应力集中区域加密网格、选择单元类型、处理接触面。MeshGraphNet的Encoder只做MLP映射——跳过了前处理中最需要工程判断的部分。

### A.4 训练/推理 = 建模/求解——最后一行才是重点

| CAE工程流程 | 深度学习工程流程 |
|---|---|
| 选择物理模型 | 选择网络架构 |
| 网格无关性研究 | 超参数调优 |
| 模型校准（对标实验） | 训练（对标数据） |
| 模型验证 | 验证集评估 |
| Garbage In, Garbage Out | Garbage In, Garbage Out |

**训练数据的V&V谁来做？** 物理AI的训练数据通常来自CFD/FEA仿真，但仿真本身就有误差。用"有误差的仿真结果"训练出的"有误差的神经网络"做预测——误差会叠加还是抵消？

### A.5 CFD视角：湍流建模——AI最想攻克但还没攻克的堡垒

CFD的核心难题是湍流建模。AI切入CFD的两种方式：

第一种：用ML修正RANS的湍流模型。但问题是：用DNS数据训练的修正模型，泛化到未见过的流动物理机制时还可靠吗？核心困难不仅是雷诺数的差异，而是物理机制的根本变化——从附着流到分离流、从稳态到转捩、从简单几何到工业复杂几何。

第二种：用神经网络直接做CFD代理模型。在论文常用的Re~100量级benchmark上训练的模型，对Re~10⁶的工业问题有多大意义？

值得注意的是，格子玻尔兹曼方法（LBM）天然适配GPU并行，计算效率比传统有限体积法高出一个量级。不只是AI在向物理靠拢，传统CFD方法本身也在向GPU原生架构演进。当CAE的求解器也变成GPU原生的时候，"神经网络比传统求解器快1000倍"这个优势还剩多少？

### A.6 系统工程视角：涌现现象——AI的优势还是陷阱？

物理AI在组件级别学到的模式，能否在系统级别涌现出正确的行为？关键区别：物理系统的涌现是由守恒定律约束的，而神经网络的涌现没有这个约束——它可能涌现出看起来合理但物理上不自洽的行为。

## 附录B：五个坑的完整踩坑记录

以下是我用PyTorch Geometric和DeepMind的MeshGraphNet数据集，从零复现流体仿真GNN预测的详细经历。

### B.1 坑一：数据集巨大

MeshGraphNet的训练数据集动辄数GB，TFRecord格式。还没开始训练，光是下载、格式转换、磁盘I/O就能卡半天。

CAE翻译：瞬态分析的.odb结果文件。

更深的问题：传统CAE可以只输出感兴趣区域、只存关键时间步。MeshGraphNet的训练需要每个时间步的全场数据。工业级问题的训练数据量将是天文数字。

### B.2 坑二：GPU显存

MeshGraphNet做全图训练——整个非结构网格必须放进GPU显存。我的T600（4GB）必须大幅缩减网格规模。

CAE翻译：图分割 = 域分解法。

更深的问题：传统域分解有成熟的数学理论保证精度。GNN的图分割？分割引入的误差有理论上界吗？

### B.3 坑三：数据格式碎片化

DeepMind用TensorFlow + TFRecord。在PyTorch里跑需要装TF只为读数据。PyG和DGL两个GNN框架API不兼容。

CAE翻译：HyperMesh的.hm、ANSA的.ansa、Abaqus的.inp——格式转换永远是日常。

更深的问题：物理AI的数据标准几乎为零。缺少标准化数据格式是物理AI无法工业化的隐性瓶颈。

### B.4 坑四：训练时间

学术复现报告（arXiv:2408.06101）：RTX 3070 Ti上训练cylinder_flow，约58小时。

更深的问题：GNN训练时间和什么成正比？目前只有经验观察，缺少理论指导。

### B.5 坑五：长程交互和泛化

最关键的问题：传统CAE基于第一性原理（守恒方程），对新几何天然泛化。GNN的泛化依赖训练数据覆盖度。GNN到底学到了"物理规律"，还是"训练数据的统计特征"？

### 挑战总结

| 物理AI的挑战 | CAE里的老朋友 | 还没解决的新问题 |
|---|---|---|
| 数据集太大 | 结果文件管理 | 工业级数据管线效率 |
| GPU显存不够 | 求解器内存不足 | 图分割的误差理论上界 |
| 数据格式碎片化 | 前处理格式兼容 | 缺少行业标准数据格式 |
| 训练时间长 | 求解收敛时间长 | 训练时间的scaling理论 |
| 长程交互不足 | 全局刚度矩阵组装 | 局部MP + 全局约束的统一框架 |
| 泛化到未见几何 | 模型适用范围 | 学到的是物理规律还是统计特征？ |

00 从投影到碎片

上一篇我提出了一个命题：AI是自然界整体交互过程的多模态投影，具有强大的预测能力，但没有自我意识。

这篇继续追问一个关键问题：投影是怎么形成的？

先看一个容易被忽略的事实。GPT-4的训练数据来自互联网上几十亿份文档——新闻、论文、代码、小说、对话、食谱。这些东西不是一个整体。每一份文档都只是自然界某次交互过程的一个局部快照——就像从不同角度、不同时间、不同语言拍下的散落照片。

核心洞见：投影本身就意味着碎片化。
每条训练数据都是自然界交互的一次局部观测，天然孤立、天然不完整。它们散落在高维概念空间的各个角落——就像分散在各地的小水坑，深浅不一，分布各异，彼此不通。

神经网络要做的，就是让这些水坑贯通。

01 神经网络是连通管道

做CAE多孔介质仿真的同行应该很熟悉PNM——孔网络模型。多孔材料里有无数孔隙，孔隙之间通过孔喉连接。流体从一端流向另一端的能力，取决于孔喉的连通状况和两侧的压力驱动。

神经网络里对应的"压力"，是信息浓度梯度：一个问题进入网络，就像在概念空间制造一个浓度高点，驱动相关知识向它汇聚。这正是注意力机制的物理本质。

与天然多孔介质有一个关键区别：岩石中的孔喉结构由地质过程决定；而神经网络中的权重矩阵由反向传播优化。训练不是自然渗流——训练是孔道工程。

多孔介质 PNM神经渗流模型 NPM
孔隙（Pore）数据碎片（局部知识单元）
孔喉导度 gik有效权重矩阵 Weff
孔隙浓度场 ci激活场 ai（信息势差驱动力）
渗流阈值 pc临界密度 ρc（涌现阈值）

主方程：L·c = q统一主方程：aout = Weff·ain

02 统一主方程：误差=0验证

从信息守恒的唯一前提出发——稳态推理时，任意内部节点流入等于流出——直接推导出：

统一主方程：aout = Weff · ain

Weff 是有效导度矩阵，三种主流架构各有对应：

架构Weff 构造特性
前馈NNW（固定权重）有向单向流
GNN ★D-½(A+I)D-½ = Lnorm无向扩散 = PNM精确对应，误差=0
Transformersoftmax(QKT/√dk)动态自适应，softmax = 流量守恒
重要说明：softmax行归一化 = 流量守恒这个性质对任意行随机矩阵都成立，并非Transformer独有。Transformer类比的更深层问题——Weff的输入依赖性在PNM中对应什么——我们推测类似于非牛顿流体的浓度依赖扩散系数。这是NPM最重要的开放理论问题。GNN与PNM的等价是精确的数学推导；Transformer的映射目前是启发性类比。

03 核心方程组

三个输入：N（数据量）、σ（分布广度 0~1）、τ（任务深度）。

数据密度： ρ = NB / exp(α·σ)   (C) 启发式
临界密度： ρc = k · σδ · τ   (B)+(C) 混合
泛化能力： Ceff = (ρ−ρc)β  (ρ>ρc)   (A) 推导
最小层数： Lmin = α·σα·τ·10   (C) 启发式
最小参数： Pmin = NB·σα   (C) 启发式
参数标注：α=1.5，β=1.4，δ=1.0，k=10 为渗流理论经验起始值，明确未拟合。(A) 由渗流理论直接推导；(B) 借用渗流文献；(C) 工程启发式，待Chinchilla和BIG-Bench数据校准。以下数值示例展示趋势方向，绝对值依赖于待拟合参数。

核心洞见：不要问"要多少数据"，要问"ρ有没有超过ρc？"

σ小（领域集中）→ 空间小 → 密度高 → 小模型够用
σ大（跨域通用）→ 空间大 → 密度低 → 必须大模型

04 涌现不是一个点，是一张场

大家通常把涌现描述成：规模到了某个临界点，能力突然出现。好像开关拨了一下。

这是严重简化的。

真实发生的事情：不同功能有不同的临界密度。随着数据量增加，水系一片一片地连通，功能一个一个地解锁。这不是开关，是水闸——不同闸口在不同水位依次打开。

能力类型临界密度Cn≈渗流含义
事实检索最低 ρc₁1.0短路径局部连通
翻译 / 摘要低 ρc₂1.3跨语言水系贯通
代码生成中 ρc₃1.8逻辑链路连通
复杂推理高 ρc₄2.5多跳路径连通
数学证明最高 ρc₅4.0全域贯通+极深路径

做FEA的人都熟悉应力云图。涌现不是"结构什么时候断裂"这个单一临界点，而是整张应力场。

为什么GPT-3能翻译但不太会数学？因为ρ超过了ρc₂，但还没超过ρc₅。不是不够聪明，是翻译水系通了，数学水系还没通。

05 污水、堵塞与假连接：数据质量的渗流视角

看似存在的水，不一定是干净的水。训练数据里有三种"坏水"，机制各不相同：

数据类型PNM类比机制症状
噪声数据 稀释剂 降低有效密度ρeff，需更多数据才能到达ρc 能力延迟，方向不偏
错误数据 污染流体 扩散方向偏转，虚假连接形成 自信地说错话（幻觉）
矛盾数据 双向抵消 同一通道正反驱动力对冲，净流量→0 前后矛盾，输出不一致
三种问题，三种解法：
噪声 → 增加数据量或缩小σ（致密化）
错误 → 方向校正（RLHF）
矛盾 → 去重与冲突消解（数据清洗）

把所有数据问题都当成"需要更多数据"，是混淆了三种物理上完全不同的故障模式。

06 三个现象的深入分析

量化压缩为什么先丢数学能力？

量化降低Weff精度 = 均匀缩小孔喉截面。首先断裂的是ρc最高的路径——支撑数学推理的最长、最脆弱的多跳路径。短路径（事实检索）更鲁棒。NPM预测退化顺序：数学→代码→翻译→事实检索。可直接对照GPTQ/AWQ论文中MMLU子类得分验证。

RAG为什么对有些任务有效，有些没用？

RAG = 临时向特定区域注水。NPM预测效果呈倒U型曲线：当模型内在ρ刚好低于任务ρc时效果最大——注水恰好推过阈值；ρ远高于ρc（已充分连通）时增益小；ρ远低于ρc（根本不连通）时注水也救不了。

微调为什么只需很少数据？

微调 = 局部致密化。将有效σ缩小到窄子域，大幅提升局部ρ。不需要全局连通，只需目标域的局部连通。PNM术语中：微调是在特定岩层钻孔，不是灌溉整个油藏。这解释了为什么LoRA用1000-10000条数据就能在窄域取得强效果。

07 这个模型能解释什么

NPM统一解释七个工程师最关心的现象：

① 为什么通用模型必须远大于专用模型？通用σ大，概念空间指数增大，同等N的密度急剧下降。

② 为什么微调只需少量数据？局部渗流，局部提高密度。

③ 为什么量化压缩先损失细粒度能力？高ρc路径最脆弱，最先断开。

④ 为什么RAG对某些任务特别有效？临时注水，对ρ刚好不够的任务效果显著。

⑤ 为什么蒸馏有效但有上限？σ压缩有下限，高ρc能力先丢失。

⑥ 为什么多语言训练需要更大模型？多语言≈更大σ。

⑦ 为什么注意力机制有效？softmax行归一化 = 流量守恒约束下的自适应导度分配。

09 三个可证伪预测

一个不能被否定的框架不是科学。NPM给出三个具体的、可测试的预测：

预测1（能力涌现顺序）：Pythia模型族（70M–12B）的能力涌现应遵循ρc排序：翻译先于代码生成先于复杂推理。可对照BIG-Bench得分验证。

预测2（RAG有效性）：RAG应在高ρc任务上提供更大增益。可对照BEIR基准验证。

预测3（量化退化顺序）：量化应先消除数学能力，后消除翻译和事实检索。可对照GPTQ/AWQ的MMLU子类得分验证。

10 诚实说明

NPM是工程模型，不是数学证明。核心指数（α=1.5，β=1.4，δ=1.0）是物理估计值，需要用Chinchilla扩展律、BIG-Bench涌现数据拟合验证。

这没有问题。CFD里的湍流模型从来没有严格证明，Moody图是纯经验拟合。工程模型的标准是：逻辑自洽、物理可解释、参数可拟合、预测有泛化性。NPM满足这个标准。

当前可信度评估
✓ 定性预测（趋势方向）可信——域越宽→σ越大→密度越低→需要更多层数
△ 定量数字（具体Ceff值）参考用——指数拟合完成前不建议直接用于工程决策
✓ 统一主方程已数值验证（GNN误差=0，注意力流量守恒通过）
✓ 三个可证伪预测已给出，待实验验证

与已有学术研究的关系：Lubana et al.（2024）用渗流解释Transformer涌现；Brill（2024）在Bethe格严格推导幂律扩展律。NPM建立在这些工作之上。诚实评估：在扩展律这个具体问题上，Brill的推导比NPM更严格。NPM的差异化贡献在于：多阈值场视角、统一主方程覆盖三种架构、PR法量化σ、工程框架定位。
